01 Problem Statement

Wordle is a popular word-guessing game where players have 6 attempts to guess a 5-letter word. After each guess, the game provides feedback: green for correct letters in correct positions, yellow for correct letters in wrong positions, and gray for incorrect letters.

The Challenge

Can we build an algorithm that consistently outperforms human players by making optimal guesses based on available information?

The key questions were:

  • How do we mathematically model the feedback system?
  • What makes a "good" next guess when multiple options exist?
  • How can we efficiently search through 12,000+ possible words?

03 Technical Challenges

Challenge 1: Regex Pattern Generation

The most complex part was converting Wordle feedback into a regex pattern that could filter the word list.

# A is wrong, R is in word but not position 2, etc.

def build_regex_pattern(guess, feedback):
    pattern = ""
    excluded_letters = set()
    required_letters = {}

    for i, (letter, color) in enumerate(zip(guess, feedback)):
        if color == "green":
            pattern += letter
        elif color == "yellow":
            pattern += f"[^{letter}]"  # Not this letter here
            required_letters[letter] = i
        else:
            pattern += "."
            excluded_letters.add(letter)

    return pattern, required_letters, excluded_letters

Challenge 2: Word Scoring Algorithm

To choose the best next guess, I developed a scoring system based on letter frequency in remaining possible words.

Python scoring.py 
def score_word(word, possible_words):
    """Score a word based on how much information it provides"""
    letter_freq = {}

    # Calculate frequency of each letter in possible words
    for possible in possible_words:
        for letter in set(possible):
            letter_freq[letter] = letter_freq.get(letter, 0) + 1

    # Score this word
    score = 0
    for letter in set(word):
        score += letter_freq.get(letter, 0)

    return score

04 Solution Approach

The final algorithm follows this decision-making process:

1
Optimal First Guess

After analyzing letter frequency across all valid words, "ARISE" emerged as the statistically best opening move, containing the 5 most common letters.

2
Pattern-Based Filtering

Convert feedback into regex patterns to eliminate impossible words. For example, if E is green in position 5, pattern becomes ....E

3
Information Maximization

Score remaining words by how many new letters they test. Prioritize words that will narrow down possibilities regardless of outcome.

4
Adaptive Strategy

As possibilities narrow, switch from exploration (testing new letters) to exploitation (guessing likely answers).

06 Lessons Learned

üéØ Data-Driven Decisions Beat Intuition

My initial instinct was to use common words like "STARE," but frequency analysis proved "ARISE" was mathematically superior. Trusting the data over gut feeling was key.

üîß Regex is Powerful for Pattern Matching

I initially tried manual string filtering, but converting feedback to regex patterns made the code cleaner and 3x faster. Learning regex deeply was worth the investment.

üß™ Test with Real Scenarios

Competing against myself revealed edge cases that unit tests missed. Real-world testing is irreplaceable for validating algorithm performance.

üìä Simple Metrics Can Be Deceptive

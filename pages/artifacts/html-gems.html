📂 <details> & <summary>

Click to reveal more goodies

Native disclosure widget with accessibility baked in—no JS needed.

  • Works with keyboard and screen readers
  • Style the open state via details[open]
HTML 
<details>
  <summary>Click to reveal more goodies</summary>
  <p>Native disclosure widget with accessibility baked in.</p>
</details>

Create expandable sections (FAQ, code blocks, etc.) with a11y-friendly behavior out of the box. Perfect for accordions without JavaScript!

✨ Interactive

💬 <dialog>

Hi there 👋

This is a native modal with focus trapping and ESC to close.
HTML 
<dialog id="dlg">
  <form method="dialog">
    <h3>Modal Title</h3>
    <p>Modal content here</p>
    <button value="ok">OK</button>
  </form>
</dialog>

<button onclick="document.getElementById('dlg').showModal()">Open</button>

Accessible modal dialogs without heavy libraries. Native focus trapping, backdrop clicks, and ESC key support. Just one line to open/close!

✨ Interactive

✏️ contenteditable + spellcheck

Click here and type. This region is editable and will underline misspellings. Try it out!
HTML 
<div contenteditable="true" spellcheck="true" aria-label="Editable note">
  Click here and type...
</div>

Turn any element into a lightweight editor without JavaScript. Perfect for note-taking interfaces, comment boxes, or simple WYSIWYG editors.

📝 Forms & Inputs

📋 <datalist>

HTML

HTML

<input type="color" value="#46c8ff" />

Native inputs with built-in validation reduce JavaScript bloat and improve accessibility. Browsers provide native pickers and validation UI.

📊 <meter>

92
78°C
HTML 
<meter min="0" max="100" low="50" high="90" optimum="100" value="92">
  92 out of 100
</meter>

Represent bounded scalar values like ratings, disk usage, or scores. Browser automatically styles based on thresholds (low/high/optimum).

📊 Indicators

⏳ <progress>

66%
HTML 
<progress value="66" max="100">66%</progress>

A semantic way to show task completion. Screen readers announce progress automatically. Omit value for indeterminate state.

📊 Indicators

🧮 <output> + Form Calculations

Total: $7.00

HTML 
<form oninput="total.value = (+price.value) * (+qty.value)">
  <input id="price" type="number" value="10">
  <input id="qty" type="number" value="2">
  <output name="total" for="price qty">20</output>
</form>

Semantic element for displaying calculation results. The 'for' attribute links it to input fields for better accessibility.

🎬 Media

🖼️ <figure> / <figcaption>

HTML 
<figure>
  <img src="photo.jpg" alt="Description">
  <figcaption>Photo caption here</figcaption>

Captions that are programmatically tied to the media, not just visually nearby. Works with images, diagrams, code blocks, and more.

🎬 Media

🎥 <video> + <track>

HTML 
<video controls>
  <source src="video.mp4" type="video/mp4">
  <track kind="captions" srclang="en" src="captions.vtt" default>

Native captions improve accessibility and searchability of media. Supports subtitles, captions, descriptions, and chapters via WebVTT files.

🎬 Media

🌅 <picture>

HTML 
<picture>

Serve the right image asset for the right viewport without JavaScript. Great for art direction and performance optimization.

⚙️ Attributes

🚫 inert Attribute

When a modal opens, you can set the rest of the page to inert to prevent focus/interaction.

HTML 
<main inert>
  <!-- This content cannot be focused or interacted with -->
</main>

Helps enforce focus trapping patterns cleanly with native support. Perfect for modal overlays and disabled sections.

⚙️ Attributes

⌨️ accesskey

Press Alt/⌥ + H to jump to top
HTML 
<button accesskey="s">Save (Alt+S)</button>

Provide quick keyboard access—use sparingly and document your keys. Can conflict with browser/OS shortcuts, so test thoroughly.
